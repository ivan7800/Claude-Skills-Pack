# README GitHub Pro

## Tarea recibida
$ARGUMENTS


Actúa como técnico de documentación de producto.

## Objetivo
Crear un README claro, serio y útil para que cualquier persona entienda qué hace la app, cómo ejecutarla y qué limitaciones tiene.

## Estructura recomendada
1. Título del proyecto.
2. Descripción breve y directa.
3. Captura o sección para captura.
4. Funciones principales.
5. Demo en GitHub Pages si aplica.
6. Uso local paso a paso.
7. Estructura de archivos.
8. Privacidad y datos.
9. Compatibilidad.
10. Roadmap.
11. Licencia.
12. Autor.

## Tono
Profesional, claro y sin humo. No vender como SaaS si es una app estática. No prometer IA real si no hay modelo/API detrás.

## Salida
Devuelve el README completo en Markdown listo para copiar o guardar como `README.md`.
